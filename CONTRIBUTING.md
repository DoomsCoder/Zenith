# Contributing to Zenith

First off, thank you for considering contributing to Zenith! It's people like you who make Zenith a great tool for everyone.

Zenith is a strict, telemetry-driven focus engine, and we welcome contributions that help improve its performance, design, or documentation.

## How Can I Contribute?

### Reporting Bugs
*   Check the [issue tracker](https://github.com/DoomsCoder/Zenith/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, a description of the bug, and steps to reproduce it.

### Suggesting Enhancements
*   Open an issue to discuss your idea before starting work. This ensures your contribution aligns with the project's philosophy.

### Your First Code Contribution
If you're looking to get started with Open Source (OSS) or Zenith, here are some great "Good First Issue" areas:
*   **Kotlin Cleanup:** Refactoring older code to use more idiomatic Kotlin or modern Coroutines patterns.
*   **UI Polish:** Improving spacing, animations, or adding subtle haptic feedback to the "Halo" dial.
*   **Documentation:** Fixing typos or adding comments to complex trigonometric logic in the Canvas components.

## Pull Request Process

1.  **Fork** the repository and create your branch from `main`.
2.  **Install** dependencies and ensure the project builds correctly.
3.  **Implement** your changes. If you've added code that should be tested, please add tests.
4.  **Format** your code following the existing project style.
5.  **Submit** a Pull Request with a clear description of what you've changed.

## Style Guide
*   Use **Kotlin** for all logic.
*   Follow **Jetpack Compose** best practices (State Hoisting, Stable types).
*   Keep the "True Black" (`0xFF0A0A0A`) aesthetic for all UI changes.
